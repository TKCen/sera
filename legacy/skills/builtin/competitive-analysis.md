# Competitive Analysis

## Purpose

Provide a structured comparison of competing options to support informed decision-making about technology choices, product positioning, or strategic direction.

## Process

### 1. Define the comparison scope

- What decision does this analysis serve?
- What are the candidate options to compare?
- What dimensions matter most for this decision?

### 2. Gather information

- Research each candidate systematically using web-search and knowledge-query.
- Prefer official documentation, independent benchmarks, and user community feedback over vendor marketing.
- Note the recency of each data point — fast-moving spaces go stale quickly.

### 3. Build the comparison matrix

Evaluate each candidate across relevant dimensions, for example:

- Feature completeness
- Performance characteristics
- Operational complexity
- Licensing and cost
- Community and support
- Integration compatibility
- Maturity and stability

### 4. Identify differentiators

- What does each option do uniquely well?
- What are the hard limitations or known weaknesses?
- What is each option's trajectory (improving, stable, declining)?

## Output Format

- Comparison table (candidates as columns, dimensions as rows)
- Narrative summary of key differentiators
- Recommendation with rationale, tied explicitly to the decision context
- Caveats: what could change this recommendation
